## Project Conventions

Clean Code, SOLID, and security-first development. Conventional Commits for all git messages.

**Detailed rules:** see `.claude/rules/` directory.

**Key principles:**
- Small functions (<30 lines), meaningful names, guard clauses over nesting
- Handle errors explicitly ‚Äî no empty catch blocks, typed errors with context
- Parameterized queries, input validation, no hardcoded secrets (OWASP Top 10)
- Atomic commits in imperative mood, `<type>[scope]: <description>` format
- Profile before optimizing ‚Äî prefer O(n) over O(n¬≤), paginate large datasets

## TypeScript Conventions

Strict TypeScript with maximum type safety. No `any`, prefer `unknown`. Discriminated unions over type assertions.

**Detailed rules:** see `.claude/rules/typescript/` directory.

**Key rules:**
- Enable `strict`, `noUncheckedIndexedAccess` ‚Äî use `satisfies` and `as const` over enums
- Explicit return types on exported functions, `catch (error: unknown)`
- Prefer `import type` for type-only imports, barrel exports only at package boundary
- Use `Result`/discriminated union patterns for expected failures

## React Conventions

Server Components by default (React 19+). Minimal `useEffect` ‚Äî derive state, use event handlers.

**Detailed rules:** see `.claude/rules/react/` directory.

**Key rules:**
- One component per file (<120 lines), composition over prop drilling
- Rules of Hooks are non-negotiable ‚Äî no conditional hooks, complete dependency arrays
- Avoid useEffect for: derived state, user events, data transforms ‚Äî use event handlers or useMemo
- Test with Testing Library: query by role/label, test behavior not implementation

## Next.js Conventions

App Router architecture. Server Components by default ‚Äî add `'use client'` only at leaf-level interactive boundaries.

**Detailed rules:** see `.claude/rules/nextjs/` directory.

**Key rules:**
- Follow route segment conventions: layout ‚Üí template ‚Üí error ‚Üí loading ‚Üí not-found ‚Üí page
- Server Actions are public endpoints ‚Äî validate ALL inputs with Zod, check auth+authorization
- Understand the 4 caching layers: request memoization, data cache, full route cache, router cache
- Use `next/image`, `next/font`, `next/link` ‚Äî never raw `<img>`, `<a>` for internal links
- Environment secrets: never prefix with `NEXT_PUBLIC_`, guard with `import 'server-only'`

## Playwright Conventions

Cross-browser E2E testing. Auto-waiting, web-first assertions, trace viewer.

**Detailed rules:** see `.claude/rules/playwright/` directory.

**Key rules:**
- Use locators: `getByRole`, `getByLabel`, `getByTestId` ‚Äî never CSS selectors
- Web-first assertions (`expect(locator).toBeVisible()`) with auto-retry
- Page Object Model for complex flows, fixtures for shared setup
- Parallel execution by default ‚Äî tests must be independent

## Docker Conventions

Multi-stage builds, minimal images, security-first container design.

**Detailed rules:** see `.claude/rules/docker/` directory.

**Key rules:**
- Multi-stage builds: separate build and runtime stages, copy only artifacts
- Use specific base image tags (not `latest`), prefer distroless or Alpine
- Non-root user (`USER`) in production, read-only filesystem where possible
- `.dockerignore` to exclude `node_modules`, `.git`, secrets from build context

## Project-Specific Notes

**TrackIt** ‚Äî local-first time tracking app. No auth, no external backend. SQLite via `better-sqlite3` (server-only).

### Stack
- Next.js 15 App Router + React 19 + TypeScript (strict, `verbatimModuleSyntax`)
- SQLite via `better-sqlite3` (server-only, `serverExternalPackages`)
- Tailwind CSS, no UI component library

### Key directories
```
app/           # Next.js routes + API handlers (GET/POST only, no Server Actions)
components/    # All UI ‚Äî each has .feature + .steps.tsx for BDD unit tests
lib/           # db.ts (SQLite singleton), types.ts, timeUtils.ts
tests/
  bdd/         # Playwright E2E feature files + step definitions
  unit/        # vitest setup (jest-dom)
```

### Testing
Two BDD layers, both using Gherkin feature files:

| Layer | Tool | Config | Command |
|-------|------|--------|---------|
| Unit  | vitest + @amiceli/vitest-cucumber + RTL | `vitest.config.ts` | `npm run test:unit` |
| E2E   | playwright-bdd + Playwright | `playwright.bdd.config.ts` | `npx playwright test --config playwright.bdd.config.ts` |

**Unit test pattern** (critical):
- Import from `@testing-library/react/pure` (disables auto-cleanup ‚Äî each step is a separate `it` block)
- Call `cleanup()` at the start of each `Given` that renders
- Store `container` in closure, query with `within(container)` ‚Äî never global `screen`
- Register `And(...)` steps explicitly; `Then(...)` does not match `And` in feature files

### Build notes
- ESLint is skipped during `next build` (`eslint.ignoreDuringBuilds: true`) ‚Äî run `npm run lint` separately
- `components/**/*.steps.tsx` and `tests/**` are excluded from the Next.js tsconfig (not type-checked at build time)
- All type-only imports must use `import type` (`verbatimModuleSyntax` enforces this)
